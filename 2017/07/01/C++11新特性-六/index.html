C++11新特性-六

  • C++11

使用C++11中便利的工具
方便的处理日期，时间，字符串相关的转换。

6.1处理日期和时间的chrono库

3种类型
时间间隔：duration
时钟：clocks
时间点：time point

6.1.1记录时长的duration

原型：
template< class Rep, class Period = std::ratio<1, 1="">>class duration;
Rep:是一个数值类型，表示时钟书的类型；第二个表示时钟周期。
时钟周期原型
template< std::intmax_t Num, std::intmax_t Denom = 1>class ratio;
表示每个时钟周期的秒数，第一个分子，第二个分母 = 1；
标准库常用时间间隔 ：时，分，秒，毫秒，微秒，纳秒
chrono 提供了获取时间间隔的时钟周期数的方法count();
时间间隔之间可以做运算
统一一种时钟规则
ratio< x1, y1>count1;
ratio< x2, y2>count2;
x1,x2的最大公约数x；
y1,y2的最小公倍数y；
统一时钟:ration< x, y>.
通过duration_cast< chrono::minutes>(t).count()计算时间；

6.1.2表示时间点的time point

表示一个时间点，用来获取从它的clock的纪元开始所经过的duration和当前的时间，还可以做时间的比较和算数运算。
typedef duration< int, std::ratio<60*60*24>> days_type;
time_point< system_clock,days_type> today = time_point_cast< days_type>(system_clock::now());

6.1.3获取系统的时钟的clocks

system_clock:代表真实世界的挂钟时间，依赖于系统。
steady_clock：不能被调整的时钟。保证先后调用的now()得到的值是不会递减的
high_resolution_clock:高精度时钟.
system_clock的to_time_t方法可以将一个time_point转换为ctime:
std::time_t now_c = std::chorono::system_clock::to_time_t(time_point);
from_time_t方法刚好相反

6.1.4计时器timer

用high_resolution_clock实现

6.2数值类型和字符串的相互转换

to_string()将任何数值类型转换成字符串类型
atoi:字符串->int类型
atol:string->long
atoll:string->long long
atof:string->浮点数

6.3宽窄字符转换

C++11增加了unicode字面量的支持，可以通过L来定义宽字符：
std::wstring str = L”中国人”;
std::wstring_convert:将款字符串转换成窄字符串；
转换器：std::convert_utf8,std::convert_utf16,std::convert_utf8_utf16

C++11的其他特性

7.1委托构造函数和继承构造函数

7.1.1委托构造函数

允许在同一类中一个人构造函数可以调用另外一个构造函数


        /使用委托构造函数不能形成环，否则会抛出异常
        使用了代理构造函数就不能用类成员初始化
        class_c(int my_max) : max(my_max){};
        只能通过成员赋值初始化
        /
        class_c(int my_max){
            max = my_max > 0 ? my_max : 10;
        }
        class_c(int my_max, int my_min) : class_c(my_max){
        //这里通过递归委托上面的构造函数完成成员变量的初始化
            min = my_min > 0 & my_min < max ? my_min : 1;
        }

7.1.2继承构造函数

需要在派生类中一一对应继承基类的构造函数，直接继承基类是不能和基类一样构造初始化的。


    struct Base{
        Base(){};
        func(){};
    };
    struct Derived : Base{
        using Base::Base;//声明使用基类构造函数
        //此方法不仅对构造函数使用，对其它同名函数也适用
        using Base::func;
        func(int a){return a;};
    };

7.2原始的字面量

可以直接表示字符串的实际含义，因为有些字符串带有一些特殊的字符，不如转义，需要做专门处理。
原始字符串字面量R “xxx(raw string)xxx”和c#中的@字符作用类似
string str = “D:\A\B\test.text”; //D:AB est.text
string str = “D:\A\B\test.text”; //D:\A\B\test.text
string str = R”(D:\A\B\test.text)”; //D:\A\B\test.text

7.3final和override关键字

c++11 新增了final关键字来限制某个不能被继承，或者某个虚函数不能被重写，和javafinal和c#sealed关键字功能类似
如果修饰函数final只能修是虚函数，并且要放到类或者函数后面。
virtual coid foo() final; //只能修饰虚函数，不能被重写
struct B final : A{
void foo(); //error
};
struct C : B{}; //error B不能被继承
override关键字确保在派生类中声明的重写函数与基类的虚函数有相同的签名，表明重写基类的虚函数
struct A{
virtual void func(){};
};
struct D : A{
//重写函数
void func() override{};
};

7.4 内存对齐

7.4.1 内存对齐的介绍

内存对齐，或者字节对齐，是一个数据类型所存放的内存地址的属性，是一个无符号整数示，必须是2^N；
当内存对齐为8，指这个数据类型所定义出来的所有变量内存地址都是8的倍数
对于struct中默认的对齐方式等于最大的成员的对齐值。
相对偏移量随内存位置而改变。

7.4.2 对内存的内存对齐

malloc：使用当前平台默认的最大内存对齐数对齐内存。
分配特定内存对齐的内存块：MSVC：_aligned_malloc
gcc: memalign

  • 前者实现

    7.4.3 利用alignas指定内存对齐大小

    基本用法：alignas(32) long long a = 0;
    #define XX 1
    struct alignas(XX) MyStruct_1{};
    template < size_t YY = 1>
    struct alignas(YY) MyStruct_2{};
    static const unsigned ZZ = 1;
    struct alignas(ZZ) MyStruct_3{};
    但其只能改大，不能改小，需要使用#pragma pack 或 _Pragma
    _Pragma(“pack(1)”)
    struct MyStruct{};
    _Pragma(“pack(0)”)
    还可以alignas(int) char c;//按照int的对齐方式对齐char

    7.4.4 利用alignof和std::alignment_of获取内存对齐大小

    alignof获取内存大小,可以用来获取变参内存大小
    alignof(xx);
    std::alignment_of补充alignof的功能，继承std::integral_constant,拥有value_type,type,value;
    //获取大小
    int alignsize = std::alignment_of::value;
    //align实现std::alignment_of
    temeplate< class T>
    struct alignment_of : std::integral_constant< std::size_t,alignof(T)>{};

    7.4.5 内存对齐的类型std::aligned_storage

    一个内存对齐的缓冲区,很多时候要分配一块单纯的内存块，在使用placement new 在这块内存上构建对象。
    template< std::size_t Len, std:size_t Align = /default-alignment/>
    struct aligned_storage;
    声明std::aligned_storage< sizeof(T), alignif(T)>;
    用法：
    struct A{};
    typedef std::aligned_storage< sizeof(A), alignof(A)>::type Aligned_A;//构造内存块
    Aligned_A a;
    new (&a) A(10, 20);

    7.4.6 std::max_align_t和std::align操作符

    std::max_align_t：用来返回当前平台的最大默认内存对齐类型。
    alignof(std::max_align_t); //当前平台的最大默认内存对齐数
    std::align:用来在一大块内存当中获取一个符合指定内存要求的地址
    std::align(alignof(int), sizeof(char), pt, space);
    在内存块中指定内存对齐为alignof(int),找一块大小为sizeof(char)的内存，放入到地址pt中，将buffer从pt开始的长度放到space中。

    7.5 C++11 新增的便利算法

    1.all_of:检查区间[first,last)中是否所有的元素都满足一元判断式p,
    any_of:检查区间[first,last)中是否至少有一个元素都满足一元判断式p,
    none_of:检查区间[first,last)中是否所有的元素都不满足一元判断式p,

    std::xx_of(v.begin(), v.end(), isEven);
    2.find_if_not算法
    查找不符合某个条件的元素
    std::find_if_not(v.begin(), v.end(), isEven);
    3.copy_if
    根据条件复制
    std::copy_if(v.begin(), v.end(), v1.begin(), {return i%2 != 0;});
    4.iota算法
    用来生成有序序列
    std::iota(v.begin(), v.end(), is); //is是序列生成规律
    5.minmax_elemen算法
    同时获取最大最小值的算法
    minmax_elemen(v.begin(), v.end());//存放到pair返回
    6.is_sorted和is_sorted_until
    is_sorted:判断一个序列是否排好序。
    is_sorted_ubtil:返回序列中前面已经排好序的部分
