C++11新特性-五

  • C++11

使用C++11让多线程开发变得简单

5.1线程

5.1.1线程的创建

std::thread t(f);
t.join();//实现线程阻塞，知道线程函数执行结束，有返回值将被忽略
t.detach();//将线程和线程对象分离，求不能在对线程进行控制

线程也可以接受任意个数的参数
std::thread t(f, 1, 2, “test”);
线程虽然不可以复制，但却可以移动
std::thread t(f);
std::thread t1(std::move(t));
通过std::bind或lambda表达式创建线程
std::thread t1(std::bind(f, 1, 2));
std::thread t2({}, 1, 2);
保证线程对象的生命周期
a. t,join();
b. t.detach();
c. std::vector< std::thread> g_list;
std::vector< std::shared_ptr> g_list2;

5.1.2线程的基本用法

1.获取当前信息
t.get_id() //获取当前线程id
std::thread::hardware3_concurrency() //获取CPU核数：CPU上能处理芯片组的数量
2.线程休眠
std::this_thread::sleep_for(std::chrono::seconds(3));

5.2互斥量

头文件
< thread>< mutex>< chrono>
是一种同步的原语，是一种线程同步的手段，用来保护多线程访问的共享数据。
std::mutex:独占的互斥量，不能递归使用
std::timed_mutex:带超时的独占互斥量，不能递归使用
std::recursive_muutex:递归互斥量，不带超市功能。
std::recursive_timed_mutex:带超时的递归互斥量

5.2.1 独占互斥量

std::mutex g_lock;
g_lock.lock();
//独占运行时
g_lock.unlock();
用lock_guard简化lock/unlock,更加安全：在构造是自动锁定互斥量，RAII技术
std::lock_guard< std::mutex> locker(g_lock); //出作用域后自动解锁

5.2.2 递归互斥量

递归锁允许同一线程多次获得该互斥锁，可以解决同一线程需要多次获取互斥量时死锁的问题
std::recursive_mutex mutex;
std::lock_guard< std::recursive_mutex> lock(mutex);
最好不要使用
多线程同步引起的晦涩问题
效率低
次数没有具体说明，超过限定会出现std::system错误

在获取互斥量超时时，可以进行其他操作

5.3条件变量

另一种用于等待的，当得到另外一个线程发来的通知或者超时，才会唤醒当前阻塞的线程。
condition_variable:配合std::unique_lock< std::mutex>进行wait等待
condition_variable_any:和mutex搭配使用，效率较前者较差，但更灵活
条件变量使用
a.拥有条件变量的线程获取互斥量
b.检查条件，分支进行
c.条件满足后通过notify_one或notify_all唤醒一个或所有等待线程
用例：实现同步队列，作为一个线程安全的数据共享区，用于线程间的数据读取，eg.半同步半异步线程池的同步队列。

5.4原子变量

std::atomic,使用原子变量就不需要互斥量来保护该变量了。相比于mutex更简洁。
计数器：



 #include < atomic>
 struct AtomicCounter{
   std::atomic< int> value;

   void increment(){
       ++value;
   }

   void decrement(){
     --value;
   }

   int get(){
       return value.load();
   }
};




5.5 call_once/once_flag的使用
保证多线程环境下某个函数仅被调用一次。once_flag作为call_once的入参。
实现：

    

        std::once_flag flag;
        void do_once(){
            std::call_once(flag, { std::cout << “goodgsger” << endl;});
        }
    



5.6 异步操作
异步操作相关：
std::future:获取线程函数返回值的类,
std::promise:包装一个值，将数据和future绑定起来，方便线程赋值,
std::package_task:包装一个可调用对象，以便异步调用.


5.6.1 future
std::future_status status;
do{
    status = future.wait_for(std::chorono::seconds(1));
    status 三种状态的判断
    std::future_status::deferred; //异步未开始
    std::future_status::timeout; //异步超时
    std::future_status::ready;  //异步结束
}while(不结束)；
获取结果三种方式：get //等待结束和得到返回值
                  wait //等待结束
                  wait_for//超时等待


5.6.2协助线程赋值
基本用法
std::promise< int> pr;
std::thread t({p.set_value_at_thread_exit(9);}, std::ref(pr));
std::future< int> f = pr.get_future();
auto r = f.get();


5.6.3可调用对象的包装类
std::package_task< int()> task({return 7;});
std::thread t1(std::ref(task));  //线程的异步调用
std::future< int> f1 = task.get_future();
auto r1 = f1.get();


线程异步操作函数async
可直接创建异步task
std::launch::async ：直接创建
std::launch::deferred ；延迟创建，直到调用future的get或wait时创建。
