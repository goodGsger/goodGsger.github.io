  • C++11

使用C++解决内存分配问题

#shader_ptr 共享的智能指针
std::shared_ptr使用引用计数，每一个shared_ptr的拷贝都指向相同的内存，在最后一个析构的时候，才释放内存。

4.1.1的基本用法

1.初始化
通过构造函数，std::make_shared辅助函数和reset方法来初始化
std::shared_ptr< int > p(new int(1));
std::shared_ptr< int > p2 = p;
std::shared_ptr< int > ptr;
ptr.reset(new int(1)); //当智能指针中有值时，调用会是引用减1.

if(ptr)
{
cout << “ptr is not null”;
}

  • 用make_shared 来构造智能指针，更高效。
  • 不可以直接赋值。
    2.获取原始指针
    std::shared_ptr< int > p(new int(1));
    int p = ptr.get();
    3.制定删除器
    当引用计数为0时，在动调用删除器
    std::shared_ptr< int > p(new int, [](int     p){delete p;});
    可用于动态数组
    也可用make_shared_array
    template
    shared_ptrmake_shared_array(size_t size)
    {
    return shared_ptr(new T[size], default_delete());
    }

    4.1.2注意问题

    a.不能用原始指针初始化多个shared_ptr；
    b.不能在函数实参中创建shared_ptr指针
    because:不同编译器的执行顺序不同 如果先执行初始化，则内存被分配，然后再执行其他参数出现异常，则会造成内存泄露
    c.通过shared_from_this() 返回this指针。不要将this作为shared_ptr返回出来。
    because:this是一个裸指针，会导致重复析构。当用同一个指针this构造两个智能指针时，会各自析构一次。
    解决：
    calss A: public std::enable_shared_from_this
    {
    std::shred_ptr     GetSelf()
    {
    return shared_from_this();
    }
    };
    std::shared_ptr     spy(new A);
    std::shared_ptr     p = spy->GetSelf();
    d.避免循环引用否则会导致内存泄漏
    由于循环引用会让两个共享指针引用计数为2，导致离开作用域后并不会变成0，导致内存泄露
    e.是不支持数组参数的。

    4.2 unique_ptr 独占的智能指针

    1.初始化
    unique_ptr myPtr(new T);
    unique_ptr myOtherPtr = myPtr;///error
  • 不可以复制给另一个unique_ptr
  • 拥有权限的转移
    unique_ptr myOtherPtr = std::move(myPtr); //ok
  • make_unique 方法实现
    支持数组初始化
    std::unique_ptrptr(new int[10]);
    2.删除器
    和shared_ptr不同，需要定义删除器类型
    std::unique_ptrptr(new int[i],{delete p;});
    lambda不能有捕获，在没有捕获的情况下可以直接转换成智能指针
    解决：用std::function 修饰
    练习：自定义unique_ptr的删除器

    4.3 weak_ptr弱引用的智能指针

    是用来监视shared_ptr的，不会是引用计数加一，他没有重载操作符* -> 只是监视shared_ptr中管理的资源是否存在
    weak_ptr还可以用来返回this指针和解决循环引用的问题

    4.3.1 weak_ptr的基本用法

    a.通过use_count()来获得当前观测资源的引用计数
    shared_ptr sp(new int(10));
    weak_ptr wp(sp);

cout << wp.use_count() << endl;
b.通过expired()方法判断所观察资源是否已释放
c.通过lock()方法来获取监视的shared_ptr

4.3.2 返回this指针

4.1.2解惑
基类中有weak_ptr用来检测this指针，调用shared_from_this()方法时，会调用lock()，将所观察shared_ptr返回

4.3.3解决循环引用问题

在循环引用时，只要将其中一个成员变量改为weak_ptr即可
利用了其不会引用加一的性质

4.4通过智能指针管理第三方库分配的内存

由于第三方库返回的指针是原始指针，所以在用完后如果没有第三方库的释放接口，就容易造成内存泄漏。
用智能指针管理第三方接口，由于用到的地方会很多所以提炼出公共函数
std::shared_ptr< void > Guard(void p)
{
return std::shared_ptr sp(p,[this]?(void p){GetHandle()->Release(p);});
}
*不可以直接 Guard(p); 访问
这样会访问野指针，它是一个右值，不赋值给一个指针，执行完后p就释放了
对于这种不安全操作，还可以用宏定义

#define GUARD(P) std::shared_ptr p##p(p, {GetHandle()->Release(p);});
也可以用unique_ptr来管理

#define GUARD(P) std::unique_ptr p##p(p, {GetHandle()->Release(p);});
