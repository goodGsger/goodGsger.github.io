#渲染路径
Unity5.0之前渲染路径有三种方式：前向渲染路径，延迟渲染路径，顶点照明渲染路径 等级上升
不过5.0之后就将顶点照明渲染路径废弃了但是可以兼容，同时推出新的延迟渲染路径。
设置渲染路径：书上说在Edit->Project setting->player->setting中设置Rendering Path 显然现在5.5中并没有，在5.5中可以在相机中直接设置。 由默认改成Use Graphics Setting
设置好后通过在Pass中用标签实现引用
即

    
     Pass{
        Tags{"LightMode"="ForwardBase"}
 }

如果没有在Pass中指定渲染路径5.x会认为是和顶点照明渲染路径等同的，很有可能出现渲染错误

##前向渲染路径
1.原理
每进行一次完整的前向渲染，就渲染该对象的图元，进行两个缓冲区的信息的计算：颜色和深度 ，其中深度设置对象是否可见，先进行深度检测，然后计算可见对象的颜色，进行更改，
2.unity中的前向渲染
有三种光照处理方式：逐顶点处理，逐像素处理球谐函数（SH）
使用的规则如下
（1）场景中最亮的平行光总是按逐像素处理的
（2）渲染模式被设置成Not Important的光源，会按逐顶点和SH处理
（3）渲染模式被设置成Important的光源，会按照逐像素处理
（4）如果实际的逐像素光照处理少于预期的，会有更多的光照按照逐像素处理方式。
