C++11新特性-一

  • C++11

类型推导

1.1.1 auto类型推导

(1)新定义：
auto x = 5;   //5为const int，推导出x为int ?为什么丢弃const；
auto pi = new auto(1);  //可以动态初始化  pi为int*
const auto *v = &x, u = 6;  &x为int*   auto为int；注意：因为在v*定义后就可以推导出auto的类型，所以紧接着定义的变量必须是对应的变量类型，初始化时要一致。
auto int r; //error 不可以再显示定义类型
auto s;  //error 必须初始化

综上：auto并不代表一个实际的类型声明，只是一个类型的"占位符"

auto原义：存储类型指示符。"具有自动存储期的局部变量"
(2)推导规则
a.当不声明为指针或引用时，auto的推导结果和初始表达式抛弃引用和CV限定符后的类型一致。
b.当声明为指针或引用时，auto的推到结果将保留初始化表达式的cv属性。
* 和函数模板参数的推导规则相似。
* auto不可以用于函数参数
(3)限制
a.不能用于函数参数
b.不能用于非静态成员变量  C++11 不支持auto非静态成员变量的初始化，但其他可以。
c.无法定义数组
d.无法推导出模板参数
template<typename T> struct Bar{};
Bar<int> bar;
Bar<auto> bar; //error 无法推导出模板参数
(4)什么时候使用
减少繁琐的变量类型申明：
map:STL
eg1. std::map<double,double> resultMap;
std::map<double,double>:: iterator it = resultMap.begin();
// auto it = resultMap.begin();
eg2. unordered_multimap
std::unordered_multimap<int,int> resultMap;
std::pair<std::unordered_multimap<int,int>::iterator,std::unordered_multimap<int,int>::iterator> range = resultMap.eaqual_range(key);
//这样的悠长的代码，怎么能显出美呢？
// auto range = resultMap.eaqual_range(key);
eg3.简化函数定义。

(5)总结
auto是一个强大的工具，且具有两面性，过多的使用会使得可读性和维护性严重下降。

1.1.2 decltype关键字
(1)获取表达式类型
decltype(exp) exp:表示一个表达式
只希望通过表达式和获取类型，而不希望和这个表达式的值相同
(2)推导规则
a.exp是标识符，类访问表达式，decltype(exp)与exp类型一直。
b.exp是函数调用，decltype(exp)和返回值类型一致
c.若是一个左值，表达式加上括号是一个左值，decltype(exp)是exp的左值引用，否则和exp类型一致。
对于纯右值，只有类类型可携带cv限定符
(3)实际应用

* 通过变量表达式抽取变量类型
vector<int> v;
decltype(v)::value_type i = 0;

1.1.3返回类型后置语法
template
auto add(T t,U u) -> decltype(t + u)
{
return t + u;
}

1.2.2模板的别名
using typedef-name = pre-name
eg. using str_map_t = std::map;
str_map_t map1;
//using 的别名语法覆盖了typedef的全部功能。
问：重定义函数指针（模板）

不需要在使用繁琐的外敷模板。虽然定义一个模板，但既不是类模板也不是函数模板，而是新的形式：模板别名。
1.2.3函数模板的默认模板参数
在C++11之前只有类函数有默认模板参数，11后函数默认模板参数也被支持。
函数的模板参数类型是根据实参自动推导得来的，并不是作为自动推导的”建议”，所以在自动推导生效时，直接忽略默认模板参数。

非聚合体：自定义构造函数

1.6.1概念和基本用法

auto f = { return a + 1};
//初始化列表不可以推导返回值类型
auto a = {return {1, 2};};
//没有参数列表是可以省略
